CASE vahendid

Mis on CASE vahendid?

CASE vahendid ehk Computer-Aided Software Engineering Tools vahendid on vahendid, mis toetavad tarkvara arendus- ja haldusprotsesse. CASE vahendid liigituvad laiemas plaanis kaheks - "ülemise taseme" CASE vahendid ehk upper CASE tools, mis on peamiselt kasutusel kasutajanõuete analüüsimiseks ja dokumenteerimiseks ning "alumise taseme" CASE vahendid ehk lower CASE tools, mis on programmi teostuseks vajalik vahend - näiteks kodeerimisel.

Mille jaoks neid kasutatakse?

CASE vahendeid kasutatakse erinevate tarkvaraprogrammide ülesehitamiseks, sh andmebaasi struktuuri genereerimiseks, testimiseks, programmeerimiseks, nende dokumenteerimiseks ja haldamiseks.

Kuidas jaotatakse case vahendeid?

  • Upper CASE vahendid
  • Lower CASE vahendid

Milliseid vahendeid saab kasutada spiraalse arendusmudeli igas elutsüklis?

Esimeses ehk planeerimise elutsüklis saab koostada dokumentatsiooni kasutades näiteks Microsoft Word programmi ning projektijuhtimiseks saab kasutada näiteks Trello programmi. Teises ehk riskianalüüsi elutsüklis saab teha visuaalse SWOT-analüüsi näiteks Canva programmis või matemaatilise riskianalüüsi Exceli programmis. Kolmandas ehk arenduse faasis saab kasutada erinevaid programmeerimise või tarkvaraarenduse programme nagu näiteks Visual Studio Code või Figma. Viimases ehk testimise faasis saab kasutada näiteks kasvõi Google Forms platvormi, mille kaudu saab koguda tagasisidet kasutajatelt.

Milliseid vahendeid olen ise juba kasutanud

Alustades lihtsamatest - olen kasutanud erinevaid Microsoft Office programme nagu näiteks Microsoft Word ja Microsoft Excel. Visuaalsetest tööriistadest olen kasutanud Canvat ja Adobe Illustratorit. Kõige uuemate vahenditena olen kasutama õppinud Visual Studio Code-i ja Githubi.
